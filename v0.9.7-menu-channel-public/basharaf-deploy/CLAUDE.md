# CLAUDE.md — قوانین پروژه «با شرف» (Claude Code این فایل را هر جلسه خودکار می‌خواند)

## ⚠️ پروتکل رله — دو اکانت نوبتی روی این پوشه کار می‌کنند

این پروژه توسط **دو اکانت Claude Code** به‌صورت نوبتی توسعه داده می‌شود. فایل `HANDOFF.md`
در ریشه‌ی پروژه «دفتر وضعیت زنده» است و تنها راه انتقال زمینه بین دو اکانت.

### در شروع هر جلسه (قبل از هر کاری — الزامی)
1. `HANDOFF.md` را بخوان: **بخش ۰ (وضعیت لحظه‌ای)** + **جدیدترین ورودی ژورنال**.
2. `git status` و `git log -5 --oneline` بزن و با ژورنال تطبیق بده.
   - اگر تغییرات commit‌نشده وجود دارد → جلسه‌ی قبل ناقص بسته شده. به کاربر بگو و
     بدون تأیید او چیزی را commit یا revert نکن.
3. به کاربر خلاصه‌ی فارسی بده: وضعیت فعلی، کار نیمه‌تمام (اگر هست)، و کار بعدی
   پیشنهادی از Backlog — بعد منتظر تأیید بمان.

### در پایان هر جلسه یا بعد از هر تغییر معنادار (الزامی)
1. یک ورودی ژورنال با قالب داخل `HANDOFF.md` به **بالای** بخش ژورنال اضافه کن
   (تاریخ، اکانت، چه‌شد، فایل‌ها، build، ناتمام، برای جلسه‌ی بعد).
2. **بخش ۰** را به‌روز کن: نسخه، تاریخ + شماره اکانت، وضعیت build، کار نیمه‌تمام،
   کار بعدی پیشنهادی.
3. اگر ژورنال > ۷ ورودی شد، قدیمی‌ها را به `project-docs/handoff-archive.md` منتقل کن.
4. `git add -A && git commit && git push` — جلسه بدون push بسته نشود.
5. اگر کاربر وسط کار جلسه را قطع کرد، قبل از خروج حتماً «ناتمام» را در بخش ۰ و
   ژورنال دقیق بنویس تا اکانت دیگر بداند از کجا ادامه دهد.

### قوانین همیشگی این پروژه
- کاربر برنامه‌نویس نیست — توضیح فارسی ساده، تصمیم‌های مهم را قبل از اجرا بگو.
- روال هر تغییر کد: `npx tsc --noEmit` (۰ خطا) → `npm run build` → ژورنال → commit/push.
- پول bigint تومان؛ تاریخ کاربری جلالی text؛ `accounts.balance` فیلد cache است
  (هر reverse جامانده = drift دائمی)؛ JWT_SECRET حتماً ≥۳۲ کاراکتر.
- قبل از دست‌زدن به منطق مالی/موجودی، بخش ۴ «مرجع پایدار» HANDOFF.md را بخوان.
- هرگز فرض نکن جلسه‌ی هم‌زمان دیگری باز نیست — اگر `git status` چیز عجیبی نشان داد، بپرس.
