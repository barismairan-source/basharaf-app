# بررسی: عنوان صفحات (page titles / metadata)

> تاریخ: ۱۴۰۵-۰۴-۰۶ (2026-06-27) · اکانت ۱ · نسخه پایه: `0.9.43`

## وضعیت فعلی

- **عنوان ریشه** (`app/layout.tsx`): `title.default = 'با شرف — سامانه حسابداری شعب'`، `template = '%s | با شرف'`.
- تقریباً **همه‌ی صفحات panel کلاینت‌اند** (`'use client'`) و چون کامپوننت کلاینت در App Router نمی‌تواند `metadata` export کند، **هیچ عنوان اختصاصی ندارند** → همه عنوان پیش‌فرض `'با شرف — سامانه حسابداری شعب'` را می‌گیرند.
- فقط ۳ layout عنوان دارند:
  - `app/order/layout.tsx` → `'سفارش بیرون‌بر'` ✅ (درست؛ با template می‌شود «سفارش بیرون‌بر | با شرف»)
  - `app/(admin)/layout.tsx` → `'Super Admin — با شرف'` ⚠️ (با template تکراری می‌شود: «Super Admin — با شرف | با شرف»)
  - root default (بالا)

## مشکلات

1. **همه‌ی ~۲۲ صفحه‌ی panel عنوان یکسان و عمومی دارند** («با شرف — سامانه حسابداری شعب»). هیچ تفکیکی بین تراکنش‌ها/انبار/گزارش‌ها نیست.
2. **admin title تکراری** («... | با شرف» دوبار).
3. صفحات عمومی (`/apply`, `/m`) عنوان اختصاصی ندارند و عنوان پیش‌فرض را می‌گیرند که **نشت internal** دارد («سامانه حسابداری شعب») — این در بخش ۳ هم پوشش داده می‌شود.

## راه‌حل

چون صفحات کلاینت‌اند، برای هر مسیر یک **`layout.tsx` سرور** کنار آن اضافه می‌شود که فقط `metadata.title` را تنظیم می‌کند و `children` را پاس می‌دهد (بدون تغییر رفتار/رندر). template ریشه خودکار «نام صفحه | با شرف» می‌سازد.

### نگاشت مسیر → عنوان (panel، سطح اول)

| مسیر | عنوان |
|---|---|
| dashboard | داشبورد |
| transactions | تراکنش‌ها |
| accounts | صندوق‌ها |
| contacts | طرف‌حساب‌ها |
| reports | گزارش‌ها |
| customers | امور مشتریان |
| reservations | رزرو میز |
| coupons | کوپن‌ها |
| employees | پرسنل |
| payroll | حقوق و دستمزد |
| recruitment | استخدام |
| inventory | انبار |
| inventory/kitchen | آشپزخانه |
| inventory/kitchen/prep | نیمه‌آماده‌ها |
| menu | مدیریت منو |
| orders | سفارش‌های بیرون‌بر |
| equipment | تجهیزات |
| purchase-orders | سفارش خرید |
| tasks | وظایف روزانه |
| settings | تنظیمات |
| logs | لاگ سیستم |

> صفحات فرزند (`[id]`, `new`, زیرصفحه‌های inventory) عنوان والد را به ارث می‌برند — معنادار و کم‌ریسک.

### اصلاحات layout موجود
- `(admin)/layout.tsx`: `'Super Admin — با شرف'` → `'مدیریت کل'` (رفع تکرار).
- عناوین عمومی (`apply`, `m`) در بخش ۳ تنظیم می‌شوند (هم عنوان درست، هم بدون نشت).

## ریسک
صفر — افزودن layout سرور که فقط children پاس می‌دهد رفتار رندر را عوض نمی‌کند. tsc/build تأیید می‌کنند.
