## ۲. جزئیات مشکلات به ترتیب اولویت (بدترین → بهترین)

### 🔴 سفارش خرید (`/purchase-orders`) — میانگین ۴.۲
**مشکل ۱:** ردیف‌های آیتم سفارش (نام + واحد + مقدار + قیمت + حذف) با `flex gap-2` در موبایل به‌شدت تنگ می‌شوند — ورودی‌های عددی کوچک‌تر از انگشت می‌شوند.
**مشکل ۲:** جدول آیتم‌های تأییدشده بدون `overflow-x-auto`.
**پیشنهاد:** ردیف‌های آیتم در موبایل به stack عمودی تبدیل شوند (`flex-col md:flex-row`). جدول در `overflow-x-auto` wrap شود.

### 🔴 مدیریت منو (`/menu`) — میانگین ۴.۲
**مشکل ۱:** جدول آیتم‌ها (`min-w-[480px]`) بدون `overflow-x-auto` — در موبایل محتوا بریده می‌شود.
**مشکل ۲:** ردیف edit inline (نام + قیمت + موجودی + toggle) در موبایل overflow می‌کند.
**پیشنهاد:** `overflow-x-auto` روی wrapper جدول. ردیف ویرایش در موبایل به grid responsive تبدیل شود.

### 🔴 حقوق و دستمزد (`/payroll`) — میانگین ۴.۳
**مشکل ۱:** دکمه‌های اقدام بالای صفحه (محاسبه، تأیید، چاپ) با `flex gap-2` در موبایل wrap نمی‌شوند.
**مشکل ۲:** جدول فیش‌ها (`grid grid-cols-4`) در موبایل بدون responsive breakdown.
**مشکل ۳:** جدول جزئیات رخدادهای حقوقی بدون `overflow-x-auto`.
**پیشنهاد:** `flex flex-wrap` برای action buttons. `grid-cols-1 sm:grid-cols-2 lg:grid-cols-4` برای گرید فیش‌ها. `overflow-x-auto` wrapper برای جداول.

### 🟡 دستور پخت (`/inventory/recipes`) — میانگین ۴.۴
**مشکل ۱:** دکمه‌های RecipeCard (چاپ، ویرایش، حذف) در موبایل stack می‌شوند بدون wrap.
**مشکل ۲:** جدول costing detail داخل کارت بدون `overflow-x-auto`.
**پیشنهاد:** `flex flex-wrap` برای button group. `overflow-x-auto` برای جدول costing.

### 🟡 کارتابل برگه‌ها (`/inventory/cartable`) — میانگین ۴.۴
**مشکل ۱:** جدول برگه‌های باز بدون `overflow-x-auto`.
**مشکل ۲:** فرم ویرایش inline — input‌های کنار هم در موبایل تنگ می‌شوند.
**پیشنهاد:** `overflow-x-auto` wrapper. `grid-cols-1 sm:grid-cols-2` برای فرم ویرایش.

### 🟡 مدیریت چک (`/cheques`) — میانگین ۴.۴
**مشکل ۱:** جدول چک‌ها بدون `overflow-x-auto`.
**مشکل ۲:** select وضعیت چک در موبایل width محدود و متن ممکن است truncate شود.
**پیشنهاد:** `overflow-x-auto` wrapper. در موبایل ستون‌های کم‌اهمیت‌تر (`hidden md:table-cell`) مخفی شوند.

### 🟡 مدیریت پرسنل (`/employees`) — میانگین ۴.۴
**مشکل ۱:** کارت employee با `grid grid-cols-4` در موبایل (نام + سمت + حقوق + آیکن‌ها) خیلی تنگ است.
**مشکل ۲:** modal ویرایش با `grid grid-cols-2 gap-3` در موبایل تنگ است.
**پیشنهاد:** `grid-cols-1 sm:grid-cols-2 lg:grid-cols-4` برای کارت‌ها. `grid-cols-1 md:grid-cols-2` برای modal.

### 🟡 تراکنش‌ها (`/transactions`) — میانگین ۴.۵
**مشکل ۱:** ۴ select فیلتر (`flex gap-3`) در موبایل کنار هم می‌نشینند.
**مشکل ۲:** متن طولانی عنوان تراکنش در ستون DataList ممکن است بدون truncate باشد.
**پیشنهاد:** `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4` برای فیلترها.

### 🟡 گزارش مالی (`/reports`) — میانگین ۴.۶
**مشکل ۱:** جدول مقایسه شعب و breakdown دسته‌بندی بدون `overflow-x-auto`.
**مشکل ۲:** ارتفاع chart‌ها (BarChart، LineChart) ثابت — در موبایل بزرگ به نظر می‌رسد.
**پیشنهاد:** `overflow-x-auto` برای جداول. `aspect-[16/9]` یا `height: {isMobile ? 200 : 300}` برای chart‌ها.

### 🟡 دستیار مالی (`/anomaly`) — میانگین ۴.۶
**مشکل ۱:** جدول یافته‌ها بدون `overflow-x-auto` — ستون «توضیح» در موبایل overflow می‌کند.

## ۵. الگوهای تکراری

### الگو ۱ — جدول بدون scroll افقی (۹ از ۱۸ صفحه)
جداول `<table>` یا `<div class="flex...">` بدون `overflow-x-auto` روی wrapper — در موبایل محتوا بریده یا overflow می‌شود. صفحات: منو، چک، کارتابل، دستور پخت، حقوق، گزارش، دستیار مالی، پرسنل، سفارش خرید.
```jsx
// الان:
<div className="bg-surface border border-border rounded-lg">
  <table>...</table>
</div>

// باید:
<div className="bg-surface border border-border rounded-lg overflow-x-auto">
  <table className="min-w-[480px]">...</table>
</div>
```

### الگو ۲ — گرید ثابت بدون responsive (۵ از ۱۸ صفحه)
`grid grid-cols-4` یا `grid grid-cols-2` بدون breakpoint — در موبایل فشرده می‌شود. صفحات: حقوق، پرسنل، داشبورد، سفارش خرید، منو.
```jsx
// الان: className="grid grid-cols-4 gap-3"
// باید: className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-3"
```

### الگو ۳ — گروه دکمه بدون flex-wrap (۳ از ۱۸ صفحه)
`flex gap-2` برای دکمه‌های action بدون `flex-wrap` — در موبایل overflow می‌کند. صفحات: حقوق، دستور پخت، منو.
```jsx
// الان: className="flex gap-2"
// باید: className="flex flex-wrap gap-2"
```

### الگو ۴ — modal form با grid ثابت (۲ از ۱۸ صفحه)
`grid grid-cols-2` در modal‌های ویرایش بدون responsive — در موبایل تنگ. صفحات: پرسنل، کارتابل.
```jsx
// الان: className="grid grid-cols-2 gap-3"
// باید: className="grid grid-cols-1 md:grid-cols-2 gap-3"
```

## ۷. نتیجه‌گیری

**مشکل سیستمی اصلی:** تقریباً همه مشکلات در یک دسته هستند — **responsive موبایل**. هیچ مشکل طراحی بنیادی وجود ندارد. ساختار، رنگ‌بندی، یکدستی و hierarchy همه عالی‌اند.

**کار باقی‌مانده برای موبایل:** ۹ مورد `overflow-x-auto` + ۵ مورد responsive grid — حدود ۳۰–۴۰ خط تغییر CSS class که می‌توانند در ۱–۲ commit کوچک انجام شوند.

**توصیه ترتیب فیکس:** Quick Wins ۱ تا ۳ (جداول، گریدها، flex-wrap) → یک commit → deploy → تأیید.
